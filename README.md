# Give Your AI a Memory

**Your AI is brilliant for about a week. Then you're re-explaining your business to it every single morning.**

This is the free fix: a short paper and two copy-paste prompts that give Claude a persistent memory of your business, so it walks into every session already knowing who you are instead of making you start over.

I'm Kyle (Dr. Pottsy). I'm not a developer. I run a wine and spirits shop, and I wired AI into the actual guts of it. This is the first thing I'd hand a fellow operator.

## What's here

- **[why-your-ai-forgets-you.md](why-your-ai-forgets-you.md)** — the paper. ~9 minutes. Why the AI forgets, the 30-minute fix, and an honest take on what it can't do. (Also readable, with a downloadable PDF, at [drpottsy.com/paper](https://drpottsy.com/paper).)
- **[prompt-a-browser.md](prompt-a-browser.md)** — paste it into Claude in your **browser** (claude.ai). It interviews you and writes you a starter "business memory." No installing anything.
- **[prompt-b-terminal.md](prompt-b-terminal.md)** — the **terminal** version (Claude Code / Cowork). Claude builds the memory right into a folder and reads it automatically every session. No more pasting.

## How to use it

1. Read the paper (or skim to the prompt that fits you).
2. Copy a prompt. Paste it as your first message to Claude. Answer its questions.
3. Save the files it gives you. From now on, Claude starts each session already knowing your business.

Start with the browser prompt if you've never touched a terminal. The paper has a section called *"The terminal isn't as scary as you think"* for when you're ready to level up.

## The honest part

This gives you the bones. What it can't do is the judgment — knowing which of *your* facts are load-bearing, finding what's actually bleeding your time and money, and keeping the whole thing from going stale. That part is the work, and it's what I do: [the Operator's Audit](https://drpottsy.com/work). But even if we never talk, run the prompt. It'll give you a morning back.
